PFO1 - Práctica Formativa Obligatoria 1 - Individual

Landing Page de Portafolio Personal (HTML y CSS)

Fechas Importantes

Lanzamiento: 25 de marzo
Entrega Final: 14 de abril

Requerimientos Generales

Estructura del Proyecto

  1. Archivo "index.html" ubicado en la raíz del proyecto.
  2. Carpeta "css" que contenga el archivo "styles.css".
  3. (Opcional) Carpeta "img" para los recursos gráficos.
  4. Archivo "README.md" que contenga una breve descripción indicando que se trata de un Trabajo Práctico (TP) y un checklist (ver detalle más adelante).

Repositorio y Publicación

  1. Crear un repositorio en GitHub y subir el proyecto.
  2. Publicar el proyecto utilizando GitHub Pages.

Privacidad

  1. Al tratarse de un enlace público, no es obligatorio publicar tu nombre y apellido completos ni utilizar una foto de perfil en la que se muestre tu rostro.
  2. Puedes optar por usar avatares, imágenes generadas con IA o fotos de tus mascotas.

Uso Obligatorio de Google Fonts

  1. Incorporar un enlace a Google Fonts en la sección "head" del documento HTML.
  2. Se deberá incluir, por ejemplo, un enlace a la fuente Nunito (u otra de tu elección) para la tipografía del proyecto.

Requerimientos HTML

Declaración y Metaetiquetas

  1. El documento debe iniciar con la declaración DOCTYPE.
  2. Usar el atributo de idioma en la etiqueta HTML (lang="es").
  3. En la sección "head" se deben incluir los siguientes elementos:
    1. Metaetiqueta de charset, por ejemplo: meta charset="UTF-8".
    2. Metaetiqueta de viewport para la correcta visualización en distintos dispositivos.
    3. Un título descriptivo (ej. "Mi Portafolio Personal").
    4. Un vínculo al archivo CSS ubicado en "css/styles.css".
    5. El vínculo a Google Fonts.

Estructura Semántica Obligatoria

  1. Sección de Encabezado (Header): Debe contener el título principal o el logo del sitio.
  2. Sección de Navegación (Nav): Debe incluir una barra de navegación con al menos tres enlaces (ej. "Sobre mí", "Proyectos" y "Contacto").
  3. Sección Principal (Main): Dividida en las siguientes sub-secciones obligatorias:
    1. Presentación Personal (id "sobre-mi"):
      • Párrafo que describa quién eres, tus intereses y tu misión como desarrollador.
      • Imagen representativa o foto (con atributo alt).
    2. Tarjetas o Columnas (id "tarjetas"):
      • Al menos dos tarjetas, cada una con imagen y texto.
      • Organizadas en columnas utilizando Flexbox o Grid.
    3. Listado de Habilidades (id "habilidades"):
      • Listado o tabla que muestre: tecnologías que dominas, tecnologías que deseas aprender y/o tus hobbies.
    4. Formulario de Contacto (id "contacto"):
      • Formulario básico con campos: Nombre, Apellido, Email y Teléfono.
      • Botón de submit.
    5. Películas Favoritas (id "peliculas"):
      • Presentar las tres mejores películas de la historia según tu criterio.
      • Cada película con título, imagen y breve descripción o comentario.
  4. Sección de Pie de Página (Footer): Incluir información adicional, como enlaces a redes sociales u otros datos relevantes.

Comentarios en el Código

  1. Insertar al menos cuatro comentarios en diferentes secciones del HTML explicando la función de cada bloque y señalando ideas de mejora a futuro.

Requerimientos CSS

  1. Selectores y Personalización:
    • Utilizar selectores basados en clases e identificadores para estilizar secciones, encabezados, botones, etc.
    • Definir una tipografía utilizando el font importado desde Google Fonts.
  2. Layout y Organización:
    • Organizar los elementos usando Flexbox o Grid, especialmente la sección de tarjetas o columnas.
    • Asegurarse de que el diseño sea responsivo usando unidades relativas (%, rem, vh) en imágenes y contenedores.
  3. Estilización de Componentes:
    • Personalizar la apariencia de tablas, botones, enlaces y formularios.
    • Ajustar dimensiones de imágenes y contenedores según sea necesario.
  4. Animaciones y Transiciones:
    • Incluir al menos una animación o transición (por ejemplo, efecto hover en tarjetas o botones).

Resumen de Puntos a Cumplir

HTML Obligatorio:

  • Declaración DOCTYPE y uso de lang="es".
  • Inclusión en "head" de metaetiquetas (charset y viewport), título descriptivo, enlace al archivo CSS y a Google Fonts.
  • Secciones obligatorias en "main": Presentación Personal (id "sobre-mi"), Tarjetas/Columnas (id "tarjetas"), Listado de Habilidades (id "habilidades"), Formulario de Contacto (id "contacto"), Películas Favoritas (id "peliculas").
  • Barra de navegación con al menos 3 enlaces.
  • Al menos 4 comentarios explicativos distribuidos en el HTML.

CSS Obligatorio:

  • Archivo "styles.css" con estilos personalizados.
  • Utilización de selectores por clases e IDs y aplicación de la tipografía de Google Fonts.
  • Organización del layout con Flexbox o Grid en la sección de tarjetas/columnas.
  • Personalización de tablas, botones, enlaces y formularios.
  • Uso de unidades relativas (%, rem o vh) para lograr responsividad.
  • Al menos una animación o transición implementada.

Repositorio y Publicación:

  • Repositorio en GitHub creado, proyecto subido y publicado mediante GitHub Pages.
  • Archivo "README.md" incluido, conteniendo la descripción del proyecto y el checklist de requerimientos.
