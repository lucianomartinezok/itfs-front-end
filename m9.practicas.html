Propuesta A

A partir del siguiente componente StyleMe, agregá estilos en línea al elemento h1 para que tenga background "lightblue" y color "darkred":

import React from "react";
import ReactDOM from "react-dom";

const StyleMe = <h1>Please style me! I am so bland!</h1>;

ReactDOM.render(StyleMe, document.getElementById("root"));

Guía de resolución

// En React, style recibe un OBJETO de JavaScript, no un string CSS.
// Las propiedades van entre llaves dobles: style={{ ... }}

const StyleMe = (
  <h1 style={{ background: "lightblue", color: "darkred" }}>
    Please style me! I am so bland!
  </h1>
);
Importante tener en cuenta:
  • El atributo es style (minúscula), no class como en HTML.
  • El valor es un objeto: style={{ clave: "valor" }} — las llaves externas son JSX, las internas definen el objeto.
  • Los nombres CSS van en camelCase: backgroundColor, fontSize, etc. (aunque background y color funcionan tal cual).
  • Los valores numéricos sin unidad (como padding: 10) se interpretan en píxeles automáticamente.

Propuesta B

Aplicar estilos en línea en React puede resultar molesto si se desea usar más que solo unos pocos estilos. Una alternativa más agradable es almacenar un objeto de estilo en una variable e inyectar esa variable en JSX.

A partir del componente StyleMe de la actividad anterior, declará una nueva constante llamada styles y asigná los estilos background "lightblue" y color "darkred" al elemento h1.

import React from "react";
import ReactDOM from "react-dom";

const StyleMe = <h1>Please style me! I am so bland!</h1>;

ReactDOM.render(StyleMe, document.getElementById("root"));

Guía de resolución

// Separás la definición de estilos del JSX.
// Esto escala mejor cuando hay muchas propiedades.

const styles = {
  titulo: {
    background: "lightblue",
    color: "darkred"
  }
};

const StyleMe = (
  <h1 style={styles.titulo}>
    Please style me! I am so bland!
  </h1>
);

// style={styles.titulo} → React lee el objeto y lo aplica al h1
Importante tener en cuenta:
  • Podés organizar estilos por elemento: styles.titulo, styles.boton, etc.
  • En JSX usás style={styles.titulo} (una sola llave) porque ya estás referenciando una variable.
  • Esta técnica es útil para pocos estilos; para apps grandes conviene CSS Modules, styled-components o frameworks como Bootstrap.
  • El objeto styles se crea una sola vez; no se recrea en cada render si está fuera del componente.

Propuesta A — Cambio de tema (claro / oscuro)

Creá un componente que permita cambiar el tema de una aplicación (por ejemplo, claro y oscuro).

Pista: podés utilizar useState para manejar el estado del tema actual.

Guía de resolución

import { useState } from "react";

function App() {
  const [tema, setTema] = useState("claro");

  const estilosTema = {
    claro: { background: "#fff", color: "#222" },
    oscuro:  { background: "#222", color: "#fff" }
  };

  const alternarTema = () => {
    setTema(tema === "claro" ? "oscuro" : "claro");
  };

  return (
    <div style={estilosTema[tema]}>
      <h1>Tema actual: {tema}</h1>
      <button onClick={alternarTema}>
        Cambiar a {tema === "claro" ? "oscuro" : "claro"}
      </button>
    </div>
  );
}
Importante tener en cuenta:
  • useState guarda el tema actual; al cambiar con setTema, React re-renderiza y aplica los nuevos estilos.
  • Podés usar un objeto de estilos indexado por tema (estilosTema[tema]) para no repetir lógica.
  • En proyectos reales, el tema se suele compartir con Context API para que todos los componentes lo lean sin pasar props manualmente.
  • El botón debe llamar a una función que invierta el valor del estado, no mutarlo directamente.
Extensión opcional: guardá la preferencia en localStorage para que persista al recargar la página.

Propuesta B — Ruta protegida con autenticación

Creá una ruta que requiera que el usuario inicie sesión antes de poder acceder a ella. Para esto, debés agregar la sintaxis de rutas protegidas y una lógica de autenticación en la aplicación.

Guía de resolución

import { BrowserRouter, Routes, Route, Navigate } from "react-router-dom";
import { useState } from "react";

// Componente "guardián": decide si muestra la ruta o redirige al login
function RutaProtegida({ autenticado, children }) {
  if (!autenticado) {
    return <Navigate to="/login" replace />;
  }
  return children;
}

function Dashboard() {
  return <h2>Panel privado — solo usuarios logueados</h2>;
}

function Login({ onLogin }) {
  return <button onClick={onLogin}>Iniciar sesión</button>;
}

function App() {
  const [autenticado, setAutenticado] = useState(false);

  return (
    <BrowserRouter>
      <Routes>
        <Route path="/login" element={
          <Login onLogin={() => setAutenticado(true)} />
        } />
        <Route path="/dashboard" element={
          <RutaProtegida autenticado={autenticado}>
            <Dashboard />
          </RutaProtegida>
        } />
      </Routes>
    </BrowserRouter>
  );
}
Importante tener en cuenta:
  • RutaProtegida envuelve la ruta privada y evalúa si autenticado es true.
  • <Navigate to="/login" /> redirige automáticamente sin renderizar el contenido protegido.
  • La autenticación puede ser un simple useState (como acá) o venir de un contexto / API real.
  • Instalá la dependencia: npm install react-router-dom y envolvé la app con BrowserRouter.
  • Usá replace en Navigate para que el usuario no vuelva atrás al dashboard con el botón "Atrás" del navegador sin estar logueado.

Creá una aplicación que muestre una lista de destinos turísticos:

  • Cada destino debe estar representado por una imagen y un nombre.
  • Al hacer clic en uno, la app navega a una ruta específica con información detallada del destino.

  • Si la ruta tiene un destino incorrecto, mostrá un componente NotFound con mensaje de error y botón para volver al inicio.

// datos/destinos.js — array con id, nombre, imagen, descripcion

Propuesta B — Restaurantes

Creá una aplicación que muestre una lista de restaurantes:

  • Cada restaurante debe estar representado por una imagen y un nombre.
  • Al hacer clic, navegá a una ruta con información detallada: menú y ubicación.
  • Si la ruta tiene un restaurante incorrecto, mostrá un componente NotFound con mensaje de error y botón para volver al inicio.

Guía de resolución — estructura sugerida

// datos/restaurantes.js
export const restaurantes = [
  {
    id: "pizza-norte",
    nombre: "Pizza Norte",
    imagen: "...",
    menu: ["Muzzarella", "Fugazzeta", "Calabresa"],
    ubicacion: "Av. Corrientes 1234, CABA"
  }
];

// pages/RestauranteDetalle.jsx
import { useParams, Link } from "react-router-dom";
import { Button } from "react-bootstrap";
import { restaurantes } from "../datos/restaurantes";

function RestauranteDetalle() {
  const { id } = useParams();
  const resto = restaurantes.find(r => r.id === id);

  if (!resto) {
    return (
      <div>
        <h2>Restaurante no encontrado</h2>
        <Button as={Link} to="/">Volver al inicio</Button>
      </div>
    );
  }

  return (
    <div>
      <h2>{resto.nombre}</h2>
      <img src={resto.imagen} alt={resto.nombre} />
      <h3>Menú</h3>
      <ul>{resto.menu.map(item => <li key={item}>{item}</li>)}</ul>
      <h3>Ubicación</h3>
      <p>{resto.ubicacion}</p>
    </div>
  );
}

// NotFound reutilizable para ambas apps
function NotFound() {
  return (
    <div className="text-center mt-5">
      <h2>404 — No encontrado</h2>
      <p>El recurso que buscás no existe.</p>
      <Button as={Link} to="/">Volver al inicio</Button>
    </div>
  );
}
Importante tener en cuenta:
  • La lógica es la misma que en destinos turísticos, pero el detalle incluye menú (array) y ubicación (string).
  • Reutilizá un componente NotFound centralizado; evitá duplicar el mensaje de error en cada página.
  • Usá Button as={Link} to="/" de React-Bootstrap para el botón de volver — combina estilos Bootstrap con navegación SPA.
  • Organizá la app en carpetas: components/, pages/, datos/ para mantener el código legible.
  • La ruta dinámica suele ser /restaurante/:id; el id en la URL debe coincidir con el del array de datos.
Comparación A vs B: La Propuesta A exige PropTypes y React-Bootstrap de forma explícita. La Propuesta B es similar pero enfatiza el detalle con menú y ubicación. Ambas comparten el patrón lista → clic → ruta dinámica → detalle o NotFound.
